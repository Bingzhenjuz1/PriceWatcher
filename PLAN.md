# PLAN.md

## 1. 项目愿景
当用户准备下单时，输入关键词即可查看淘宝/天猫、京东、拼多多的当前候选价格，并得到“当前最低价”建议，帮助用户以更优价格购买商品。

## 2. 当前 User Story（已确认）
- 当网上购物时，我希望能够看到我关注商品在主流购物网站上的最低价，这样我可以以当时最优惠的价格下单。

约束与口径：
- 输入方式：关键词搜索
- 覆盖平台：淘宝/天猫、京东、拼多多
- 价格口径：预估到手总价
- 搜索策略：搜索时实时抓取；日常列表允许缓存延迟
- 返回规模：每个平台 Top 3 候选
- 展示字段：平台名、商品标题、当前价、链接
- 同款匹配：品牌 + 型号 + 规格规则匹配
- 优先目标：准确性优先于速度

## 3. MVP 范围
In Scope:
- Web 页面关键词搜索
- 三平台结果聚合
- 最低价计算与展示
- 日常列表缓存与后台刷新

Out of Scope:
- App 客户端（后续阶段）
- 高级价格趋势分析
- 复杂权限与多租户

## 4. 里程碑
### M1：工程与最小搜索链路
- 初始化 Next.js + TypeScript + Prisma
- 建立搜索接口与统一结果数据结构
- 完成至少 1 个平台适配器 + mock 其余平台
- 前端可输入关键词并展示聚合结果

### M2：三平台聚合与同款匹配
- 接入三平台适配器（官方接口优先）
- 实现同款匹配规则（品牌/型号/规格）
- 实现最低价计算与异常价格过滤
- 支持单平台失败降级

### M3：缓存刷新与可交付质量
- 引入 Redis 缓存（列表可延迟）
- BullMQ 后台刷新任务
- 补齐测试（单测/集成/最小 E2E）
- 完成 lint/test/build 通过与交付文档

## 5. 任务拆解（按优先级）
### P0（立即）
- 选定初始目录结构与环境变量规范
- 初始化工程脚手架
- 定义核心类型：SearchQuery / ProductCandidate / PriceResult
- 实现搜索编排层（并发 + 超时 + 降级）
- 实现最低价聚合逻辑（先不做复杂匹配）

### P1
- 三平台适配器（官方/开放接口优先）
- 品牌+型号+规格匹配打分
- 缓存读写策略（搜索实时优先，列表缓存优先）
- 基础结果页与最低价卡片

### P2
- 异常值过滤与稳健性优化
- 后台刷新任务与失败重试
- 测试覆盖扩展与发布前核验
